# Degrain — История разработки

**Плагин:** Degrain — Granular Delay
**Тип:** Аудио-эффект (Stereo In / Stereo Out)
**Форматы:** VST3, AU (Audio Unit)
**Технологии:** JUCE 8, C++17, WebView UI (HTML/CSS/JS)
**Платформа:** macOS

---

## 1. КОНЦЕПЦИЯ И ИДЕАЦИЯ (2026-02-13)

### 1.1 Исходная идея

Degrain задумывался как **гранулярный дилей** — аудио-эффект, в котором сигнал захватывается в буфер задержки и воспроизводится в виде перекрывающихся гранул (коротких фрагментов звука). Ключевое отличие от обычного дилея — звук не просто повторяется с затуханием, а **дробится на частицы**, которые трансформируются с каждым проходом обратной связи.

### 1.2 Творческий бриф

**Концепция звука:**
- **Кристальный** — частотно-зависимый размер гранул создаёт стеклянные верхние частоты и тёплые низкие
- **Хрустящий** — квадратная огибающая + высокая обратная связь + коллизии гранул
- **Эволюционирующий** — хвост дилея трансформируется, а не просто затухает
- **Объёмный** — реверсивный микс гранул создаёт временную глубину, стереоразброс — пространственную ширину

**Схема сигнального пути:**
```
Вход → Буфер задержки → Движок гранул → [Диффузия] → [Стереоразброс] → [Детюн] → Выход
              ↑                 ↓                                                    |
              ↑           [Реверс-микс]                                             |
              ↑                                                                      |
              └──── HP/LP Фильтр ← Мягкий клиппер ← Мутация гранул ←──────────────┘
                                      (путь обратной связи)
```

### 1.3 Уникальные особенности (USP)

Три функции, которых нет у конкурентов:

1. **Мутация гранул (Grain Mutation)** — хвост дилея трансформируется с каждым проходом обратной связи. Гранулы уменьшаются в размере, питч постепенно дрейфует, плотность увеличивается — звук буквально «degrains» (разрушается).

2. **Частотно-зависимый размер гранул (Freq Spread)** — высокие частоты получают маленькие гранулы (кристаллический шиммер), низкие — большие (тёплые, длительные). Это ядро «кристального» характера звука.

3. **Реверсивный микс гранул (Reverse Mix)** — каждая гранула независимо решает: воспроизводиться вперёд или назад. При 50% получается эффект «зеркального эха».

### 1.4 Параметры (19 штук)

На этапе идеации были определены 5 групп параметров:

| Группа | Параметры |
|--------|-----------|
| **Delay** | delay_time, delay_sync_toggle, delay_sync_division |
| **Grain** | grain_size, grain_rate, grain_rate_sync_toggle, grain_rate_sync_division, grain_shape |
| **Character** | feedback, diffusion, stereo_spread, detune, low_cut, high_cut |
| **Degrain** | reverse_mix, grain_mutation, freq_spread |
| **Global** | freeze, dry_wet |

### 1.5 Варианты использования

- **Саунд-дизайн**: создание текстур, раньшеров, импактов из простых источников
- **Эмбиент**: длинный дилей + диффузия + стереоразброс + фриз = иммерсивные пады
- **Экспериментальная/глитч музыка**: короткие гранулы + квадратная форма + высокая FB + мутация
- **Электронная музыка**: темп-синхронизированные ритмические паттерны
- **Живые выступления**: фриз + управление параметрами для скульптурирования гранул в реальном времени

## 2. ИССЛЕДОВАНИЕ DSP (2026-02-13 — 2026-02-14)

Перед разработкой было проведено детальное исследование алгоритмов DSP, которые требуются для реализации плагина.

### 2.1 Кольцевой буфер задержки

**Ключевое решение:** размер буфера — степень двойки (2²⁰ = 1 048 576 сэмплов), что позволяет использовать битовую маску вместо операции деления по модулю для быстрого чтения/записи.

**Интерполяция Эрмита (кубическая):**
Для чтения на нецелых позициях используется кубический интерполятор Эрмита — он обеспечивает плавный звук без артефактов при питч-шифтинге и немонотонном чтении.

### 2.2 Движок гранул

**Пул из 64 гранул:** все гранулы предварительно выделены в памяти — нет выделений на аудио-потоке. При нехватке слотов применяется «похищение голоса» (voice stealing).

**Оконные функции:**
- **Хэнн (Hann/Sine):** `0.5 × (1 − cos(2π × phase))` — гладкая огибающая, нет щелчков
- **Треугольник:** линейный нарастание/спад
- **Квадрат:** плоская вершина с мягкими краями (1% длины) — агрессивный, «хрустящий» звук
- **Пила (Ramp):** `1.0 − phase` — ударная, перкуссивная атака

### 2.3 Диффузия Шрёдера

Четыре каскадных all-pass фильтра (фильтры Шрёдера) с простыми задержками ([113, 337, 601, 1009] сэмплов при 48 кГц) создают размытие гранул, напоминающее ранние отражения реверберации.

### 2.4 Обратная связь и мягкий клиппер

**Мягкий клиппер (tanh):** предотвращает неконтролируемое нарастание амплитуды при обратной связи > 100%. Функция линейна в пределах ±1.0 и плавно ограничивает значения выше.

**DC-блокатор:** однополюсной ФВЧ (R = 0.995, ~5 Гц) предотвращает накопление постоянной составляющей в петле обратной связи.

### 2.5 Темп-синхронизация

Чтение BPM из хост-плеера DAW через `juce::AudioPlayHead`. 18 делений такта: от 1/1 до 1/32T (прямые, пунктирные, триоли). Формула: `delay_ms = (60.0 / BPM) × 4.0 × multiplier × 1000`.

## 3. АРХИТЕКТУРНОЕ ПРОЕКТИРОВАНИЕ (2026-02-14)

### 3.1 Спецификация компонентов

Перед написанием кода была создана детальная архитектурная спецификация (`architecture.md`) со структурами данных, алгоритмами и математическими формулами для каждого компонента.

**Ключевые архитектурные решения:**
- Один движок гранул для всего сигнала (не разделять по частотным полосам)
- Диффузия применяется **после** сумматора гранул (не к каждой гранул е отдельно)
- Отдельный буфер мокрого сигнала (`wetDelayBuffer`) для чёткого разделения пути обратной связи
- Для детюна используется случайный разброс по ±cents — это создаёт более богатый хорус-эффект, чем фиксированный сдвиг
- Фриз работает в двух режимах: «чистый» (нет записи в буфер) при Mutation=0% и «живой» (обратная связь продолжает писать) при Mutation>0%

### 3.2 Оценка сложности

Плагин получил оценку сложности **5.0/5.0 (ВЫСОКАЯ)**:
- 19 параметров
- 14 DSP-компонентов (4 простых + 6 средних + 4 сложных)
- 4 уникальные особенности (+3.5 балла)

Выбрана стратегия: **поэтапная реализация** (Phase-Based Implementation) — 10 фаз с тестированием после каждой.

### 3.3 Спецификация параметров

Для каждого из 19 параметров был определён (`parameter-spec.md`):
- APVTS ID, тип (Float/Bool/Choice), диапазон, значение по умолчанию
- Коэффициент скоса (skew) для логарифмических параметров: delay_time и grain_size (0.3), grain_rate (0.3), low_cut и high_cut (0.25)
- Тип UI-элемента и цветовое оформление по секциям

## 6. РЕАЛИЗАЦИЯ — ФАЗА 4.1: ЯДРО DSP (Stage 2, Phase 4.1)

**Цель:** Аудио проходит через плагин с базовым дилеем и воспроизведением гранул.

### Кольцевой буфер задержки
- 2 буфера по `std::vector<float>` размером 2²⁰ = 1 048 576 сэмплов на канал
- Бит-маска `kBufferMask = kBufferSize - 1` для быстрого вычисления позиции
- Кубическая интерполяция Эрмита: функция `hermiteInterp(ym1, y0, y1, y2, frac)`

### Движок гранул
- Пул из 64 гранул (`std::array<Grain, 64>`)
- Планировщик: `grainAccumulator` накапливает дробные сэмплы до порога `sampleRate / grainRateHz`
- 4 оконные функции: Hann (гладкая), Triangle (линейная), Square (c мягкими краями 1%), Ramp (перкуссивная)
- Voice stealing: выбирается зерно с наибольшей фазой (ближайшее к завершению)

### Сухой/влажный микшер
- `juce::SmoothedValue<float, Linear>` с 20 мс рампой — без «ступенчатых» артефактов

**Подключённые параметры:** delay_time, grain_size, grain_rate, grain_shape, dry_wet

## 7. РЕАЛИЗАЦИЯ — ФАЗА 4.2: ОБРАТНАЯ СВЯЗЬ И ХАРАКТЕР (Stage 2, Phase 4.2)

**Цель:** Добавить петлю обратной связи, пространственную обработку и тембральное формирование.

### Путь обратной связи
- Мягкий клиппер (tanh): `softClip(x)` — линейно до ±1.0, мягкое ограничение выше
- DC-блокатор: `DCBlocker` (R = 0.995), блокирует постоянную составляющую
- `FourPoleFilter` (24 dB/окт): каскад двух `TwoPoleFilter`, каждый — два `OnePoleFilter`
- feedbackHP и feedbackLP — в петле обратной связи
- wetHP и wetLP — применяются к влажному сигналу перед dry/wet миксом
- Сглаженный коэффициент обратной связи (0–115%)

### Диффузионная сеть (Шрёдер)
- 4 каскадных all-pass фильтра (`AllpassStage`) с задержками [113, 337, 601, 1009] сэмпл (при 48 кГц)
- Коэффициент all-pass пропорционален параметру Diffusion
- Применяется к суммарному влажному сигналу перед сухим/влажным миксом

### Стереоразброс
- Каждая гранула получает случайную позицию панорамы (детерминированный хеш Кнута)
- Equal-power panning: `panL = cos(angle)`, `panR = sin(angle)`

### Детюн
- На каждую гранулу: случайный питч-сдвиг в пределах ±cents через хеш
- Скорость воспроизведения: `pow(2, cents/1200)` — нецелочисленное чтение через интерполяцию

**Подключённые параметры:** feedback, diffusion, stereo_spread, detune, low_cut, high_cut

## 8. РЕАЛИЗАЦИЯ — ФАЗА 4.3: УНИКАЛЬНЫЕ ОСОБЕННОСТИ (Stage 2, Phase 4.3)

**Цель:** Реализовать 3 уникальных особенности (USP) + фриз и темп-синхронизацию.

### Реверс-микс гранул
- Per-grain решение на основе хеша: `hash(grainSeedCounter) < reverseMix × UINT32_MAX`
- Реверсированная гранула читает буфер назад: позиция = `startReadPos − fractionalPos`
- Огибающая при этом всегда воспроизводится вперёд

### Мутация гранул
- `passCountBuffer` — буфер `uint8_t` того же размера, что и буфер задержки: отслеживает номер прохода обратной связи для каждого сэмпла
- При spawne гранулы: читается passCount для позиции чтения
- Эффекты мутации (пропорционально `mutationAmount × passCount`):
  - Размер гранулы сокращается до 20% от базового
  - Питч дрейфует на ±200 центов
  - Компенсация энергии: `amplitude / sqrt(densityScale)`

### Частотно-зависимый размер (Freq Spread)
- `BandSplitter`: два `OnePoleFilter` делят сигнал на 3 полосы (< 250 Гц, 250–2500 Гц, > 2500 Гц)
- `BandEnergy`: экспоненциальный RMS (~300 мс окно) для каждой полосы
- Масштаб размера гранулы: низкие × 6, высокие × 0.1 (100% Freq Spread)

### Система фриза
- `freezeSnapPos`: позиция снимка буфера — обновляется каждый сэмпл, когда фриз выключен (`writePos − delaySamples`)
- Когда фриз включается, `freezeSnapPos` фиксируется — все новые гранулы читают из этой точки
- 10 мс кроссфейд (freezeGain, freezeTargetGain, freezeFadeRate)
- Поддержка MIDI: Note On — включить фриз, Note Off — выключить

### Темп-синхронизация
- BPM читается из `AudioPlayHead::CurrentPositionInfo`
- `smoothedBPM` (Multiplicative SmoothedValue, 50 мс): сглаживает изменения темпа
- 18 делений такта: `kDivisionMultipliers[18]` (от 1/1 до 1/32T)
- Независимая синхронизация для delay_time и grain_rate

### Архитектурное изменение: fixed-snapshot грани

В ходе первых тестов в DAW обнаружена проблема: гранулы читали буфер относительно текущей позиции записи. Это нарушало работу фриза и мутации.

**Решение (DSP rework, 2026-02-24):** `spawnGrainAdvanced()` принимает `startReadPos`, предварительно вычисленный вызывающим кодом как `(writePos − delaySamples + kBufferSize) & kBufferMask`. Гранулы читают из фиксированного снимка: `startReadPos + fractionalPos` (абсолютная позиция). Новая функция `readBufferHermiteAbs()` обеспечивает чтение по абсолютной позиции.

**Подключённые параметры:** reverse_mix, grain_mutation, freq_spread, freeze, delay_sync_toggle, delay_sync_division, grain_rate_sync_toggle, grain_rate_sync_division

## 12. ПРОБЛЕМЫ ПОСЛЕ DAW-ТЕСТИРОВАНИЯ (2026-02-24)

После первого запуска плагина в DAW обнаружено 7 проблем, потребовавших переработки:

| # | Проблема | Тип |
|---|----------|-----|
| 1 | Delay: изменение delay_time не сдвигало «момент истории» для гранул | Архитектурная |
| 2 | Freeze: зацикливал участок аудио вместо смазывания одного момента | Переработка |
| 3 | Фильтры 6 dB/окт, применялись только в FB-пути | Апгрейд |

### Исправления DSP-переработки (2026-02-24)

**1. Архитектура fixed-snapshot гранул:**
Гранулы теперь читают из фиксированной позиции снимка (`startReadPos`), вычисленной как `(writePos − delaySamples + kBufferSize) & kBufferMask`. Изменение delay_time явно сдвигает «исторический момент», который читают гранулы.

**2. Фриз — новое поведение (smear):**
`freezeSnapPos` обновляется каждый сэмпл, когда фриз выключен. При активации фриза `freezeSnapPos` фиксируется. Все новые гранулы спавнятся в этой точке, создавая «размытие» (smear) единственного замороженного момента.

**3. Фильтры 24 dB/окт + на влажном сигнале:**
`TwoPoleFilter` (12 dB/окт) заменён на `FourPoleFilter` (24 dB/окт, каскад). Добавлены `wetHP` и `wetLP` — применяются к влажному сигналу до dry/wet микса. Фильтры теперь слышны немедленно.

**4. Усиленные эффекты:**
- Мутация: гранула сжимается до 20% (было 40%), дрейф питча ±200 центов (было ±50)
- Freq Spread: низкочастотные гранулы до 6× длиннее (было 3×), высокочастотные — до 10% (было 35%)
- Реверс: архитектурно исправлен — реверсированные гранулы читают снимок назад

**5. Dropdown UI:**
`.combo-dropdown` изменён с `top: 100%` на `bottom: 100%` — открывается вверх, не выходит за границы.

## 13. РАУНД 4 ИСПРАВЛЕНИЙ: ФОРМУЛА ОТОБРАЖЕНИЯ JS (2026-03-04)

### Обнаруженная ошибка

В JavaScript-коде UI для отображения значений параметров (ручки показывают «250 ms») использовалась формула на основе натурального логарифма:

```javascript
// НЕВЕРНО (использовался натуральный логарифм):
normToValue = (norm) => Math.exp(norm × Math.log(max/min)) × min
```

Однако JUCE `NormalisableRange` с коэффициентом skew использует формулу степени:

```
value = min + (max − min) × norm^(1/skewFactor)
```

**Результат ошибки:** при norm = 0.5 ручка delay_time показывала ~45 мс вместо реального значения ~199 мс (дефолт 250 мс соответствует norm ≈ 0.535).

### Исправление

```javascript
// ВЕРНО — соответствует формуле JUCE NormalisableRange:
function normToValue(paramId, norm) {
    const info = paramDisplayInfo[paramId];
    if (info.skew) {
        if (norm <= 0) return info.min;
        return info.min + (info.max - info.min) * Math.pow(norm, 1.0 / info.skew);
    }
    return info.min + norm * (info.max - info.min);
}

// Skew коэффициенты:
const paramDisplayInfo = {
    delay_time:    { min: 1,    max: 2000,  unit: 'ms',  skew: 0.3  },
    grain_size:    { min: 5,    max: 500,   unit: 'ms',  skew: 0.3  },
    grain_rate:    { min: 0.1,  max: 100,   unit: 'Hz',  skew: 0.3  },
    low_cut:       { min: 20,   max: 5000,  unit: 'Hz',  skew: 0.25 },
    high_cut:      { min: 200,  max: 20000, unit: 'Hz',  skew: 0.25 },
    output_gain:   { min: -24,  max: 12,    unit: 'dB'              },
};
```

**Дополнительно исправлено:** единица измерения `output_gain` имела ведущий пробел (`' dB'`), что давало двойной пробел в отображении. Исправлено на `'dB'` с добавлением знака `+` для положительных значений.

**Результат:** pluginval strictness 5 — пройден. Сборка VST3 и AU — чистая.

## 15. РАУНД 5 ИСПРАВЛЕНИЙ: ЩЕЛЧКИ И ВЫПАДЕНИЯ ЗВУКА (2026-03-04)

После тестирования в DAW обнаружены периодические выпадения звука (плагин на долю секунды прекращает работу) и цифровые щелчки. Проведена глубокая аудит DSP-кода.

### Обнаруженные и устранённые проблемы

**1. КРИТИЧНО: BPM → NaN — главная причина выпаданий**

`smoothedBPM` является `SmoothedValue<Multiplicative>`. При остановке воспроизведения в DAW хост возвращает BPM = 0. При вызове `setTargetValue(0)` множитель становится `pow(0/current, 1/N) = 0`. На следующем блоке: `pow(target/0) = NaN`. Далее `static_cast<int>(NaN)` — неопределённое поведение → случайный доступ к памяти → выпадение звука.

**Исправление:**
```cpp
// NaN-безопасная проверка (NaN >= 20 возвращает false):
if (!(bpm >= 20.0f)) bpm = 120.0f;
// Гарантируем ненулевой таргет для Multiplicative SmoothedValue:
smoothedBPM.setTargetValue(juce::jmax(20.0f, static_cast<float>(currentBPM)));
```

**2. Переполнение delaySamples**

При медленном темпе (20 BPM × деление 1/1D × 96 кГц) расчётная задержка = 1 728 000 сэмплов > `kBufferSize` = 1 048 576. Чтение за пределами буфера → UB → щелчок.

**Исправление:**
```cpp
const float delaySamples = juce::jmin(
    (delayTimeMs * 0.001f) * sr,
    static_cast<float>(kBufferSize - 4)
);
```

**3. Voice stealing — щелчок при похищении зерна**

Старая логика выбирала зерно с наибольшей фазой. Но зерно с квадратной огибающей при phase = 0.85 имеет амплитуду 1.0 — внезапное обрывание → щелчок.

**Исправление:** выбирается зерно с наименьшим текущим значением огибающей `applyWindow()`:
```cpp
float lowestEnv = std::numeric_limits<float>::max();
for (int i = 0; i < kMaxGrains; ++i) {
    float env = applyWindow(g.phase, g.windowType, g.lengthSamples,
                            static_cast<int>(fractPos));
    if (env < lowestEnv) { lowestEnv = env; slotIndex = i; }
}
```

**4. NaN в wetDelayBuffer**

NaN, попавший в буфер (например, из-за ошибки BPM), циркулирует в петле обратной связи бесконечно.

**Исправление:** проверка перед записью в буфер:
```cpp
if (!std::isfinite(fbWriteL)) fbWriteL = 0.0f;
if (!std::isfinite(fbWriteR)) fbWriteR = 0.0f;
```

**5. outputLimiter допускает ±1.25**

Функция `tanh(x × 0.8) / 0.8` имеет асимптоту ~1.25 (не 1.0). DAW жёстко обрезает значения выше 1.0 — слышный щелчок.

**Исправление:**
```cpp
outL = juce::jlimit(-1.0f, 1.0f, outL);
if (!std::isfinite(outL)) outL = 0.0f;
```

**6. Прыжок коэффициента диффузии (allpass)**

Параметр Diffusion менялся между блоками без сглаживания → резкое изменение коэффициента all-pass фильтра → скачок на выходе → щелчок.

**Исправление:** добавлен `smoothedDiffusion` (Linear SmoothedValue, 20 мс):
```cpp
smoothedDiffusion.reset(sampleRate, 0.020);
// В processBlock:
smoothedDiffusion.setTargetValue(diffusionAmt);
processDiffusion(wetL, wetR, smoothedDiffusion.getNextValue());
```

**Результат после всех исправлений:** pluginval strictness 5 — пройден. Плагин переустановлен. Выпадений звука и щелчков нет.

```

   │      └─→ readBufferHermiteAbs(startReadPos + fractionalPos)
```
